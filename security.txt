CURRENT STATE (DEVELOPMENT):
* The CV Builder relies on the public API of latexonline.cc (https://latexonline.cc/compile).
* This is strictly for initial UI testing and proof-of-concept.

THE SECURITY RISK:
* Sending raw CV data (names, emails, phone numbers, work history) to a third-party, public server violates standard data privacy practices. 
* We do not control the retention policies of the public latexonline.cc server.

THE TARGET FIX (PRODUCTION REQUIREMENT):
* Before launching to public users, we must migrate away from the public API.
* Action: Provision a private cloud server (e.g., AWS EC2, DigitalOcean Droplet).
* Action: Self-host the open-source latex-online Docker container on our private infrastructure.
* Action: Reroute the React/Node fetch requests to point exclusively to our internal compiler endpoint.

STATUS: Open

# Security Considerations & Future Improvements

This file tracks security and privacy enhancements to be implemented in the future.

1. **Auto-Deletion Policy for CVs:**
   Currently, generated CVs are saved to the backend disk indefinitely. A future improvement should implement an auto-expiration policy (e.g., deleting CVs from the server 24 hours after generation or upon user logout) to ensure data minimization and protect user privacy.

2. **Encryption at Rest:**
   PDF files (uploaded and generated) are currently stored in plain text on the server's filesystem. To protect sensitive personal identifiable information (PII) such as phone numbers, emails, and employment history from potential server breaches, AES-256 encryption should be implemented before writing files to disk, with decryption occurring only in memory during authenticated downloads.
